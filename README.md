# 쉬운말 기술사전 📖

비개발자를 위한 한국어 IT 용어집. 어려운 기술 용어를 **식당·도서관·택배 같은 일상의 비유**로 풀어 설명합니다.
(영문 사이트 [nontechnical.dev](https://nontechnical.dev/)에서 영감을 받아, 한국어 콘텐츠와 독자적인 따뜻한 에디토리얼 무드로 새로 만들었습니다.)

## ✨ 특징

- **108개 용어, 8개 주제** — 기초 다지기 · AI는 이렇게 작동해요 · AI로 만들기 · 코드와 협업 · API와 연결 · 출시와 운영 · 개발자처럼 생각하기 · 그 밖에 ([nontechnical.dev](https://nontechnical.dev/)의 구성 참고)
- **풍부한 용어 페이지** — 한 줄 정의(TLDR) · 일상 비유 · 핵심 개념 섹션 · "이걸로 할 수 있어요" · 마무리 한 방 · 관련 용어 링크
- **즉시 검색** — 용어/영문/정의/비유/설명을 한 번에 검색
- **오늘의 한 단어** — 날짜에 따라 매일 바뀌는 추천 용어
- **빌드 불필요** — 순수 HTML/CSS/JS 정적 사이트 (GitHub Pages에 그대로 배포)

## ➕ 용어 추가하기

`assets/js/terms.js`의 배열에 항목을 추가하면 끝입니다.

```js
{
  id: "고유-id",
  term: "용어 (한글)",
  en: "English Name",
  category: "기초 다지기",     // 8개 주제 중 하나 (새 주제를 쓰면 필터 칩 자동 생성)
  tldr: "한 줄로 끝내는 쉬운 정의",
  analogy: "누구나 아는 일상의 비유 한 줄",
  sections: [                  // 핵심 개념 (1~3개)
    { h: "펀치라인형 소제목", p: "2~4문장 설명" }
  ],
  unlocks: ["이걸로 할 수 있는 일", "..."],  // 선택
  punchline: "기억에 남는 마무리 한 줄",      // 선택
  related: ["관련-용어-id", "..."]            // 관련 용어 링크
}
```

주제(`category`)는 배열에 처음 등장한 순서대로 필터 칩이 만들어집니다.
`related`에 적은 id의 용어는 상세 모달에서 클릭 가능한 링크로 이어집니다.

## 🚀 GitHub Pages 배포

이 저장소는 아직 GitHub에 연결되어 있지 않습니다. 나중에 연결한 뒤:

1. GitHub에 저장소를 만들고 푸시합니다.
2. **Settings → Pages** 로 이동합니다.
3. **Source** 를 `Deploy from a branch` 로 두고, 배포할 브랜치와 `/ (root)` 폴더를 선택합니다.
4. 잠시 후 `https://<사용자명>.github.io/<저장소명>/` 에서 사이트가 열립니다.

> 별도의 빌드 과정이 없으므로 추가 설정 없이 바로 서빙됩니다.
